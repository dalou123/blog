    # webpack基础配置

    # 一、基础

    # 1.安装webpack

    首先我们先创建一个目录

    $ mkdir webpack-test && cd webpack-test
$ npm init -y   //npm 初始会自动生成package.json配置

    # 3.webpack打包

    命令行执行

    $ npx webpack
    1

    打包成功后，dist的文件夹下新增main.js文件。再浏览器上打开index.html，会显示 "Hello Webpack"， 关于main.js的产生webpack4在没有配置的情况下，它会使用默认的配置 在内存中的存储方式不同

    • 将src/index.js作为入口起点(也就是entry选项)
    • 将dist/main.js作为输出(也就是output选项)

    webpack4以下的版本（3.x，2.x，....）则需要配置文件

    # 二、使用配置文件

    # 1.webpack.config.js

    复杂的项目需要我们在webpack.config.js文件上进行相应的配置；

    // webpack.config.js  基础配置
const path = require('path')
module.exports = {
   entry: './src/index.js', //入口处为
   output: { 
       filename: 'test.js',  // 出口文件
       path: path.resolve(__dirname, 'dist')//文件所在地址
   }  
}

    # 3.webpack构建本地服务器

    构建本地服务器，浏览器监听到代码修改，自动刷新显示修改后的结果，webpack配置基于node.js的构建，需要安装依赖：

    npm install --save-dev webpack-dev-server
    1

    配置文件webpack.config.js

    const path = require('path')
module.exports = {
   entry: './src/index.js', //入口处为
   output: { 
       filename: 'test.js',  // 出口文件
       path: path.resolve(__dirname, 'dist')//文件所在地址
   },
  devServer: {
    port: 3000 //监听端口 默认8080
    contentBase: "./public", //本地服务器所加载的页面所在的目录
    historyApiFallback: true, //不跳转
    inline: true //实时刷新
  } 
}

    # 三、资源管理

    Loaders是webpack重要模块之一。通过使用不同的loader，webpack可以调用外部的脚本或工具，实现对不同格式的文件的处理，比如分析转换scss为css，或者把ES6，ES7转换为现代浏览器兼容的JS文件，对React的开发而言，合适的Loaders可以把React的中用到的JSX文件转换为JS文件

    # 1.加载CSS

    • style-loader和css-loader

    从js模块中导入一个css文件, 比如你想在index.js中引入一个css文件:

    // index.js
import './style.css'

// 或者用require()的方式
const style = require('./style.css')
    1
    2
    3
    4
    5

    你需要在项目中(也就是module配置中), 安装并添加这两个loader:

    style-loader css-loader

    $ npm i --save-dev style-loader css-loader
    1

    且在webpack.config.js中进行配置:

    const path = require("path");
module.exports = {
  entry: "./src/index.js",
  output: {
    filename: "test.js",
       path: path.resolve(__dirname, "dist"),
  },
  module: {
    rules: [
      {
        test: /\.css$/,
        use: ["style-loader", "css-loader"],
      },
    ],
  },
};

    # 2.加载图片

    • file-loader

    使用file-loader可以让我们在js和css中引入一些静态资源, 同样的, 你要先安装配置它:

    $ npm i --save-dev file-loader
    1

    配置webpack.config.js

    const path = require("path");
module.exports = {
  entry: "./src/index.js",
  output: {
    filename: "test.js",
       path: path.resolve(__dirname, "dist"),
  },
  module: {
    rules: [
      {
        test: /\.css$/,
        use: [
              "style-loader",
              "css-loader"
          ]
      },
      { 
        test: /\.(png|svg|jpg|gif)$/, 
        use: [
             'file-loader'
          ]  
      }
    ]
  }
};

    # 四、插件（Plugins）

    插件（Plugins）是用来拓展Webpack功能的，它们会在整个构建过程中生效，执行相关的任务。 要使用某个插件先通过npm安装它，然后要做的就是在webpack配置中的plugins关键字部分添加该插件的一个实例（plugins是一个数组）

    常用插件：HtmlWebpackPlugin 生成一个自动引用你打包后的JS文件的新index.html 安装

    npm install --save-dev html-webpack-plugin
    1

    更新webpack的配置文件

    const HtmlWebpackPlugin = require('html-webpack-plugin');
....
....
plugins: [
        new HtmlWebpackPlugin({
            template: __dirname + "/src/index.tmpl.html"//new 一个这个插件的实例，并传入相关的参数
        })
    ],

    Hot Module Replacement 修改组件代码后，自动刷新实时预览修改后的效果，需要进行两个配置

    • 在webpack配置文件中添加HMR插件；
    • 在Webpack Dev Server中添加“hot”参数；
      devServer: {
        contentBase: "./public",//本地服务器所加载的页面所在的目录
        historyApiFallback: true,//不跳转
        inline: true,
        hot: true
    },
  plugins: [
        new webpack.BannerPlugin('版权所有，翻版必究'), //防盗版插件，在编译的js加入该注释语句
        new HtmlWebpackPlugin({
            template: __dirname + "/app/index.tmpl.html"//new 一个这个插件的实例，并传入相关的参数
        }),
        new webpack.HotModuleReplacementPlugin()//热加载插件
    ],

    # 优化插件

    OccurenceOrderPlugin :为组件分配ID，通过这个插件webpack可以分析和优先考虑使用最多的模块，并为它们分配最小的ID UglifyJsPlugin：压缩JS代码； ExtractTextPlugin：分离CSS和JS文件

    OccurenceOrder 和 UglifyJS plugins 都是内置插件，无需安装，只要安装ExtractTextPlugin

    npm install --save-dev extract-text-webpack-plugin
    1

    webpack配置

    const ExtractTextPlugin = require('extract-text-webpack-plugin');
...
...
plugins: [
        new webpack.BannerPlugin('版权所有，翻版必究'),
        new HtmlWebpackPlugin({
            template: __dirname + "/app/index.tmpl.html"
        }),
        new webpack.optimize.OccurrenceOrderPlugin(),
        new webpack.optimize.UglifyJsPlugin(),
        new ExtractTextPlugin("style.css")
    ],
